<objective>
benchmark.sh currently trusts `is_model_cached()` (line 230), which only checks that
`$HF_CACHE/models--<repo>/snapshots` exists and is non-empty. This produces FALSE "Cached"
reports for partially-downloaded models (confirmed: `Qwen/Qwen3-14B-MLX-4bit` reports Cached
with only an 84 KB index file vs. its 7.85 GB expected size).

This plan:
1. Adds `verify_model_complete()` — real shard-completeness verification (index weight_map →
   every shard present + non-zero resolved blob; or single-file fallback).
2. Extends the pre-fetch loop so incomplete models are (re)downloaded and re-verified, with
   continue-on-failure for models that stay broken.
3. Adds a `params=` key to every candidate block and prints a model detail table
   (full HF id | params | quantization | on-disk size | approx memory) during pre-fetch.

Purpose: the benchmark sweep must never time a half-downloaded model, and the operator should
see a clear inventory of what each candidate actually is before the sweep runs.

Output: modified `benchmark.sh` and `config/candidates.conf`.
</objective>
