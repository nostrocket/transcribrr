<objective>
Refactor the three existing MLX pipeline scripts so they run **fully unattended** via flags instead of interactive `read -p` menus, and so each emits a machine-parseable `OUTPUT_FILE=<path>` line. This is the enabling vertical slice: after this plan, a user can run each sub-script non-interactively from the command line, which is the precondition for the orchestrator (plan 02) to chain them.

Purpose: The existing scripts block on `read -p` model/style prompts, making unattended pipeline execution impossible. D-01 chose per-script flag parsing as the mechanism. D-04 chose `OUTPUT_FILE=` emission for stage chaining (avoids duplicating each script's label-sanitization in the orchestrator).

Output: Three modified scripts (`transcribe.sh`, `cleanup-transcript.sh`, `summarize-transcript.sh`), each accepting model/style flags, defaulting silently to the README-recommended values, and printing `OUTPUT_FILE=<path>` after a successful write.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → sub-script | User-supplied `--model`/`--style` values and the positional file path enter each script's argument parser |
| `--model` value → output filename | A raw HF ID containing `/` could inject a path separator into the constructed output filename |
| `--model`/`--style` value → MLX/Python invocation | Flag values are passed to mlx-whisper / mlx-lm via environment/args |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `--model` value with `/` reaching the output filename (`*_summary_*.md` etc.) | mitigate | Apply the canonical sanitizer (`sed 's/mlx-community\///' \| sed 's/[^a-zA-Z0-9.-]/_/g' \| tr upper lower`) to MODEL_LABEL for any `/`-containing value in all three scripts; label is used in the filename, raw ID only passed to the model loader (ASVS V5) |
| T-01-02 | Tampering | Unknown/unexpected flag values | mitigate | Validate friendly labels and styles against fixed allow-lists; exit 1 on anything outside the list (no silent fallthrough to shell) |
| T-01-03 | Injection | Positional file path with shell metacharacters | mitigate | All file paths and flag values are referenced with double-quoted expansions; no `eval`; no unquoted variable in any command (ASVS V5) |
| T-01-04 | Tampering | Output written before success (partial/empty OUTPUT_FILE= claim) | mitigate | `OUTPUT_FILE=` line emitted only on the success path, after the write/heredoc completes |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | Phase 1 installs zero new packages (RESEARCH.md "No Package Legitimacy Audit Required"); sub-scripts' existing `.venv` auto-install is unchanged by this plan |
</threat_model>
