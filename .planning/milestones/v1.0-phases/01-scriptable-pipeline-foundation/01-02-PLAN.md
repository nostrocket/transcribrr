<objective>
Create `transcribrr.sh` — the canonical single-command orchestrator (D-05) that takes a **local MP3** and drives `transcribe.sh` → `cleanup-transcript.sh` → `summarize-transcript.sh` fully unattended. This is the Walking Skeleton: the thinnest end-to-end working slice of the project.

Purpose: This completes the Phase 1 user capability — one command instead of four hand-run interactive steps. It depends on plan 01 having made the sub-scripts non-interactive and OUTPUT_FILE=-emitting.

Output: `transcribrr.sh` at repo root, plus `SKELETON.md` recording the architectural backbone for Phase 2.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → orchestrator | User-supplied MP3 path and flag values enter `transcribrr.sh`'s parser |
| orchestrator → sub-scripts | Flag values and captured intermediate paths are passed as arguments to child scripts |
| sub-script stdout → orchestrator | The `OUTPUT_FILE=<path>` line is parsed and used as the next stage's input path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering / Injection | MP3 path or flag value with shell metacharacters reaching a sub-script invocation | mitigate | Every expansion double-quoted; values passed only as positional/flag arguments; no `eval`; no unquoted command construction (ASVS V5) |
| T-02-02 | Spoofing | Captured `OUTPUT_FILE=` path is empty or points to a non-existent file (stage failed silently) | mitigate | After each capture, verify non-empty AND `[ -f "$path" ]` before using it as next-stage input (Pitfall 1); ERR trap + `set -euo pipefail` abort on sub-script non-zero exit |
| T-02-03 | Elevation of Privilege | Path traversal via input file argument | mitigate | Preflight requires `[ -f "$MP3_FILE" ]` (regular file must exist); intermediates derive paths from the sub-scripts, not from raw user concatenation |
| T-02-04 | Denial of Service | Heavy MLX work started before a trivially-detectable failure (missing dep/file) | mitigate | `preflight_check` runs before any stage, failing fast with named cause (D-10, ROB-01) |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Phase 1 installs zero new packages (RESEARCH.md "No Package Legitimacy Audit Required"); orchestrator does not install anything and does not duplicate the sub-scripts' `.venv` bootstrap |
</threat_model>
