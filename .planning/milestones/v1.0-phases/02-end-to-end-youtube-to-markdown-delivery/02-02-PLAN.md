<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| captured metadata -> final markdown | Video title/channel/etc. (attacker-influenceable) are written into a file and a filesystem path |
| temp file -> final .md (filesystem) | A consumer reading the final `.md` must never observe a partial document |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | partial/corrupt final markdown read as complete | mitigate | Assemble into a `mktemp` temp file and atomic `mv` into place; the final `${SAFE_TITLE}.md` exists only after all stages (incl. assemble) succeed (ROB-03). EXIT trap removes the temp file on premature exit. |
| T-02-06 | Elevation of privilege | final filename derived from video title | mitigate | `FINAL_MD_PATH` uses `SAFE_TITLE` (already sanitized in Plan 01 with the `[^a-zA-Z0-9._-]` allow-list on both URL-title and local-basename paths) and is anchored to `$(pwd)/`, preventing path traversal / overwrite outside CWD via crafted titles. |
| T-02-07 | Information disclosure | metadata fields embedded verbatim | accept | Title/channel/URL are written as markdown text via `printf` (no shell re-expansion); content is the user's own requested video metadata. No PII beyond what the user chose to download; low-value local tool. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this phase. No `## Package Legitimacy Audit` required per RESEARCH.md. |
</threat_model>
