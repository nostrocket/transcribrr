<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI arg -> transcribrr.sh | The positional argument (URL or path) is untrusted external input |
| YouTube network -> yt-dlp -> filesystem | The downloaded video title (used to derive `WORK_DIR`/paths) is attacker-influenceable untrusted data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | URL passed to yt-dlp | mitigate | URL is passed as the final double-quoted literal argument `"$URL"`; no `eval`; yt-dlp treats it as a data argument, not shell. `--` is implicit since the URL is positional-last and quoted. |
| T-02-02 | Elevation of privilege | `WORK_DIR`/paths derived from video title | mitigate | `SAFE_TITLE` sanitization (`sed 's/[^a-zA-Z0-9._-]/_/g'` allow-list) strips `/` and shell metacharacters before the title is used in any path; `WORK_DIR` is anchored to `$(pwd)/` (absolute), preventing path traversal via crafted titles. |
| T-02-03 | Denial of service | untrusted-network failure / missing fields | mitigate | metadata `mapfile` guard (`${#META[@]}` < 6 -> named exit), download existence guard, and the `$CURRENT_STAGE` ERR trap fail fast with a named stage instead of proceeding on bad data. |
| T-02-04 | Tampering | playlist URL fans out to many downloads | mitigate | url-check stage rejects `[?&]list=`/`playlist` URLs before any network call; `--no-playlist` is also passed to yt-dlp as defense-in-depth. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this phase — only `yt-dlp` (a preflight-checked system binary the user installs via brew). No `## Package Legitimacy Audit` required per RESEARCH.md. |
</threat_model>
