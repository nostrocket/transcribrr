<objective>
Add the hardware-awareness and acquisition layer to benchmark.sh: detect total unified memory (HW-01), compute the 75% usable ceiling, apply the per-candidate fit gate that skips unfit models before execution with a logged reason (HW-02/03), gate disk space before any download (D-09), detect HF cache and pre-fetch only uncached fitting candidates (D-08), and download+cache the default benchmark audio sample (BENCH-06) — supporting both a URL `--sample` (download+cache) and an existing-local-file `--sample` (use directly). Purpose: Timing fidelity (no download latency in RTF — Pitfall #1 / D-08) and OOM avoidance (fit gate, HW-02/03) both depend on this layer running fully before any inference. The disk gate prevents a partial multi-GB download from failing mid-fetch.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| candidates.conf model_id → hf download / cache path | Config-supplied model ID used to build cache dir name and as download arg |
| --sample URL/path → yt-dlp | Caller-supplied sample source crosses into a network download command |
| HF Hub → local cache | Remote model weights downloaded into the local cache volume |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | model_id → HF cache dir name (path traversal) | mitigate | Cache name built via `sed 's|/|--|g'` (slashes flattened, not a path); model_id always double-quoted; only ever appended under fixed `$HF_CACHE` |
| T-04-05 | Tampering | model_id as `hf download` argument (command injection) | mitigate | `"$HF_CLI" download "$model_id"` — fixed binary, model_id always quoted, never eval'd; combined with parse-not-source from 04-01 |
| T-04-06 | Denial of Service | disk/memory exhaustion from oversized fetch/run | mitigate | The disk-space gate (D-09) and the memory fit gate (HW-02/03) ARE the mitigations — both hard-abort/skip before resource commitment |
| T-04-07 | Spoofing/Tampering | remote sample + model weights (supply chain) | accept | hf download only fetches files into a sandboxed cache (does not execute them); sample is a fixed user-chosen video; local-only tool, ASVS L1 |
| T-04-08 | Tampering | --sample value → yt-dlp | mitigate | Value double-quoted as a single yt-dlp positional arg; local-path branch uses `[ -f ]` before use; no shell expansion of the value |
</threat_model>
