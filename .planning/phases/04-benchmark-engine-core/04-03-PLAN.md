<objective>
Build the per-candidate execution engine inside benchmark.sh: a `run_candidate()` function that runs a discarded warm-up subprocess (BENCH-03), then a separate timed subprocess wrapped in `/usr/bin/time -l` for honest whole-process peak RSS (BENCH-02/04), derives the stage-appropriate speed metric (RTF for whisper, tok/s for cleanup/summarize — BENCH-04), captures a real output excerpt via the `OUTPUT_FILE=` contract (BENCH-05), writes a per-candidate JSON result (D-15), enforces a cool-down between candidates (D-14), prints live progress (BENCH-08), and continues the sweep on any single-candidate failure (D-16). Stage scripts remain unchanged — reused via `--model` + `OUTPUT_FILE=`.

Purpose: This is the measurement core. The subprocess-per-candidate (MLX Metal memory release), warm-up/timed separation, and `2>"$TIME_OUT"` temp-file split are the verified landmines that make the numbers correct and the sweep OOM-resistant.
Output: Modified benchmark.sh with the reusable run_candidate engine and JSON result writers (the per-stage loop that calls it is added in plan 04-04).
</objective>
