<objective>
Wire the full staged interactive sweep in benchmark.sh (D-01): create the per-run results directory, then for each stage in order (whisper → cleanup → summarize) run every fitting candidate through `run_candidate()` (writing a skip-result for unfit candidates), display each candidate's speed/peak-memory and a real output excerpt, prompt the human to pick the single best output, and carry that pick forward as the input to the next stage. The whisper input is the cached sample; the cleanup input is the selected transcript; the summarize input is the selected cleaned text. Finally write `sweep_meta.json` and print where Phase 5 will read results to write `settings.conf`.

It is the last plan, so it also wires the skip-JSON path from the fit gate (04-02) and the run_candidate calls from the engine (04-03).

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-13 | Tampering | selection index input | mitigate | `read -r` + strict integer regex + [1..N] bounds check; invalid input re-prompts rather than indexing out of range |
