<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user/script editor → config/settings.conf | settings.conf is user/machine-edited; transcribrr.sh reads it on every normal run (untrusted input crosses into the orchestrator) |
| orchestrator → stage scripts | resolved model strings flow into `--model "$X"` arguments of the stage scripts |
| stage exit status → user-facing error | a failed model load must translate to an actionable message without leaking sensitive paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering / Elevation | settings.conf read in transcribrr.sh | mitigate | Parse-not-source (D-02): values extracted via `grep "^KEY=" | tail -1 | cut -d= -f2-`; the file is NEVER `source`d/`eval`d/`.`-sourced, so a malicious `WHISPER_MODEL_DEFAULT=$(rm -rf ~)` is treated as an opaque string. Verify-gate asserts no source/eval of SETTINGS_FILE. Anchored `^KEY=` (Pitfall 2) prevents prefix-collision matches; keys are the three fixed literals (no user-controlled key regex). |
| T-03-03 | Tampering | model string → `--model "$X"` argument | mitigate | The resolved model value is passed as a single quoted argument `--model "$WHISPER_MODEL"`; it is never interpolated into an eval'd command string. The stage scripts treat a `*/*` value as a literal HF ID and sanitize it for filenames; a crafted value cannot break out of the argument. No shell re-expansion (matches the existing printf-not-heredoc discipline noted at transcribrr.sh line 491). |
| T-03-06 | Information Disclosure | CFG-03 error message | mitigate | The D-10 message names only the model ID and the literal string `config/settings.conf` (a relative, non-sensitive path) plus the `--benchmark` hint — it does not echo `$SCRIPT_DIR`, absolute paths, or environment. |
| T-03-07 | Denial of Service | set -e swallowing stage failures | mitigate | Pitfall 1: using `if ! STAGE_OUT=$(_run_*)` rather than `) || {...}` ensures a non-zero stage exit is actually observed and translated, instead of the pipeline silently continuing past a failed stage. |

No package installs in this plan — package legitimacy gate (T-03-SC) is N/A for Phase 3 (RESEARCH `## Package Legitimacy Audit`).
