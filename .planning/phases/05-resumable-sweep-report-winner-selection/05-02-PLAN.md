## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| candidate label / model id → write_settings_key → config/settings.conf | A label (ultimately from candidates.conf, refreshable by a skill in Phase 6) is written to a config file later parsed by transcribrr.sh |
| machine-written config (settings.conf, picks.json) → bash reads | Must be parsed, never sourced (arbitrary-code-execution risk from tampered config) |
| SIGINT during settings.conf write | Partial/corrupt config would break the next normal run |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | settings.conf atomic write integrity | mitigate | Same-directory mktemp + mv = atomic OS rename; SIGINT leaves file whole-or-absent (criterion #6); temp registered in _BENCH_TMPFILES EXIT trap |
| T-05-02 | Tampering/Injection | write_settings_key key/value | mitigate | Pass key/value via sys.argv (not heredoc interpolation); validate key against the 3 allowed keys and value against `^[A-Za-z0-9._/-]+$` |
| T-05-03 | Elevation of Privilege | reading settings.conf / picks.json | mitigate | Parse with grep/`"$PYTHON" -c` json.load — NEVER `source`; rejects code execution from a tampered config |
| T-05-07 | Tampering | predictable temp name / symlink race in config/ | mitigate | `mktemp` random suffix (`.settings_tmp_XXXXXX`) in a dir the user owns; mv overwrites atomically; no fixed temp name |
| T-05-SC | Tampering | package-manager installs | accept | No new packages installed this phase (RESEARCH: stdlib only); no install surface |
