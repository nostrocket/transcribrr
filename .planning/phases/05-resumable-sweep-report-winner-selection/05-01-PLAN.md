<objective>
Create `benchmark_helpers.py` — the standalone Python helper (stdlib only, run via `.venv/bin/python`) that owns the two text-heavy, bash-3.2-hostile capabilities of Phase 5: the cross-model transcript **divergence view** (RPT-04/RPT-05) and the **comparison report** (RPT-01, both terminal ASCII table and `report.md`).

Purpose: bash 3.2 cannot safely do sequence alignment, column wrapping, or structured-data generation. RESEARCH verified `difflib.SequenceMatcher` + `textwrap.fill` against real transcript data (turbo=75 / small=283 / distil=293 outliers on a 71-min sample). This plan delivers that mechanism as a clean CLI the bash side will wire in Wave 2 (05-03). It touches ONLY the new file — no `benchmark.sh` changes — so it runs in parallel with 05-02.

Output: `benchmark_helpers.py` with two argparse subcommands (`divergence`, `report`), all terminal output to stderr, `report.md` written to the run dir.
</objective>

    Per `label:filepath` argument: split on the FIRST `:` only (filepaths may contain none but labels never contain `:`); validate the label matches `^[A-Za-z0-9._-]+$` (reject with exit 2 otherwise — V5 input validation, prevents control-char/escape injection into terminal rendering); `os.path.isfile()` guard before reading (skip + stderr warning if absent, per D-A2/Pitfall 7); read with `errors='replace'` so binary/control bytes cannot corrupt the terminal.

Exit codes: 0 success, 1 a transcript could not be read (non-fatal — bash continues), 2 bad arguments.
